# DNF增幅预测器 - 设计文档

**日期：** 2026-04-07
**版本：** 1.0

## 项目概述

开发一个基于Web的DNF手游增幅预测工具，通过贝叶斯概率更新算法分析徽章合成结果，预测最佳增幅时机。

### 核心功能
- 用户选择增幅目标（+6→+7 到 +11→+12）
- 系统指定对应成功率的徽章合成任务
- 用户记录每次徽章合成结果
- 系统实时计算好运概率并预测增幅时机
- 提供清晰的按钮式交互界面

### 技术栈
- **前端框架：** React + Vite
- **UI框架：** Tailwind CSS
- **部署平台：** Vercel（免费）
- **数据存储：** localStorage
- **算法：** 贝叶斯概率更新

## 核心算法：贝叶斯概率更新

### 算法原理

假设游戏的随机系统存在短期"运气状态"，通过观察徽章合成结果来动态更新好运概率。

### 数学公式

**贝叶斯更新公式：**
```
P(好运|观察数据) = P(观察数据|好运) × P(好运) / P(观察数据)
```

### 具体实现

**1. 初始状态：**
```
好运概率 = 50%
坏运概率 = 50%
```

**2. 徽章成功时更新：**
```
似然比(成功|好运) = 实际成功率 × 1.3
似然比(成功|坏运) = 实际成功率 × 0.7

P(好运|成功) = 似然比(成功|好运) × P(好运) /
              [似然比(成功|好运) × P(好运) + 似然比(成功|坏运) × P(坏运)]
```

**3. 徽章失败时更新：**
```
似然比(失败|好运) = (1 - 实际成功率) × 0.7
似然比(失败|坏运) = (1 - 实际成功率) × 1.3

P(好运|失败) = 似然比(失败|好运) × P(好运) /
              [似然比(失败|好运) × P(好运) + 似然比(失败|坏运) × P(坏运)]
```

**4. 运气衰减机制：**
```
每垫3次徽章，好运概率衰减5%（防止过度依赖历史数据）
最小衰减到40%（确保不会永远不振）
```

**5. 预测决策：**
```
好运概率 > 75% → "✅ 强烈建议现在增幅！"
好运概率 > 65% → "✅ 可以增幅了"
好运概率 > 55% → "⚠️ 可以考虑，建议再垫1-2次"
好运概率 ≤ 55% → "❌ 继续垫手，运气还未回升"
```

### 交互流程

1. **初始状态：** 显示增幅目标选择按钮
2. **选择增幅目标后：** 显示对应徽章信息和成功率
3. **点击徽章结果按钮：** 实时更新好运概率和建议
4. **达到建议条件：** 高亮显示"可以增幅了"
5. **增幅后记录：** 保存实际结果用于验证

## 附录：算法示例

假设用户要增幅+6→+7（理论成功率60%），用6→7徽章垫（成功率60%）：

```
初始：好运概率 = 50%

第1次徽章：成功
- 似然比(成功|好运) = 0.6 × 1.3 = 0.78
- 似然比(成功|坏运) = 0.6 × 0.7 = 0.42
- P(好运|成功) = 0.78 × 0.5 / (0.78 × 0.5 + 0.42 × 0.5) = 65%
- 结果：好运概率 = 65% → "✅ 可以增幅了"
```

这个示例展示了算法的基本工作流程。
